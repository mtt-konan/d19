## 本次工作摘要

引入侧边排除定理：平面内若一点到单位正方形三个顶点的距离均为有理数，则该点不在正方形的任何边或边的延长线上（x=0、x=1、y=0、y=1）。这一结论由椭圆曲线方法证明。在参数化框架内，侧边条件可精确表示为整数等式，直接在内层循环中过滤，无需额外开销。

---

## 主要改动

- `src/rational_distance/search.py`
  - `_search_triple_vec`：在 numpy mask 阶段增加侧边排除条件
    - `x = 1` iff `a*p == b*r`（精确整数比较，无舍入）
    - `y = 1` iff `a*q == b*r`（同上）
    - `x = 0` / `y = 0` 在参数化下不可能（a,b,p,q,r > 0），无需显式处理
  - `_search_triple_int`（Python int 回退路径）：同样增加等价判断
  - 过滤从条件性（仅 min_rational=4 时启用）改为**无条件**，简化逻辑
- `tests/test_search.py`
  - 新增 `test_brute_force_excludes_sides`：验证所有结果中 x ∉ {0,1} 且 y ∉ {0,1}

---

## 关键决策

**为什么无条件过滤**：定理适用于所有"至少三个有理距离"的情况，搜索三顶点时同样成立。无条件过滤减少分支逻辑，结果更干净（不再出现 x=1/y=1 的边界点）。

**整数等式而非浮点比较**：侧边条件 x=1 ↔ ap/(br) = 1 ↔ ap = br，全部是整数乘法，不引入任何精度问题。这与整个搜索框架保持一致（所有判断均为整数运算）。

**不影响 brute_force_search**：暴力枚举路径中侧边点天然被 skip（Fraction 比较直接排除 x∈{0,1}），无需额外修改。
