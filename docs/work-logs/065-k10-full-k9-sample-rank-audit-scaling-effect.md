## 主结论 1：`rank ≤ 4` 在新出现的 K_10 上继续成立

这是本轮最直接的结论。

wl060 之前只跑到了 `K_8 / K_7 / K_6`。现在把真正的 `K_10` 全部补上之后，最高秩仍然只有
`4`，而且 6 个 `K_10` 里有 5 个只有 `rank = 3`。

换句话说：

```text
k 从 8 升到 10，并没有带来 rank 从 4 升到 5 或更高。
```

这使得“高 `k` 但低秩”的现象更强了，而不是更弱。

## 主结论 2：`deficit = k - rank` 在高 k 处继续拉大

```text
K_9 : deficit = 5 或 6
K_10: deficit = 6 或 7
```

和 wl060 的 `K_6 / K_7 / K_8` 相比，这一档又明显往上推了一层：

```text
K_6 : deficit 约 2-3
K_7 : deficit 约 3-4
K_8 : deficit 约 4-5
K_9 : deficit 约 5-6
K_10: deficit 约 6-7
```

因此“`k` 增长得比秩快得多”已经不是个别样本，而是非常稳定的层级现象。

## 主结论 3：高 `k` 的核心原因，不是新独立解变多，而是放大倍数把有理平方点清成整数

这部分比“`rank ≤ 4` 仍成立”更重要。

把本轮 16 个样本按公因子约去，归并到 primitive（约去公因子的底型）之后，只剩下 4 类：

```text
primitive    primitive 原始 k     本轮观测到的放大后 k       rank
────────────────────────────────────────────────────────────────────
(25, 91)            2                 9, 10                  3
(70, 117)           1                 9, 10                  3
(91, 990)           1                 9, 10                  4
(221, 704)          1                 9, 10                  3
```

也就是说，本轮看到的这些 `K_9 / K_10` 顶点，并不对应很多“新的高秩曲线类”；
它们大多只是**同少数几个低秩底型的不同放大版本**。

### 为什么放大倍数不改秩，却能改 `k`？

若 `(A, B) = (d a, d b)`，则两条曲线

```text
E_{A,B}:   Y² = X(X + A²)(X + B²)
E_{a,b}:   y² = x(x + a²)(x + b²)
```

通过

```text
X = d² x,
Y = d³ y
```

在有理数域上同构，所以**秩不变**。

但整数 `N` 的个数不是这个同构下的不变量。

如果 `N` 让 `(d a, d b)` 成为 multi-`N` 对，那么令 `n = N / d`，就得到 primitive 底型上的
有理解；反过来，只要 primitive 底型上某些有理 `n` 的分母能被 `d` 清掉，它们就会在放大后
变成新的整数 `N`。

所以：

```text
同一条低秩曲线
→ 在 primitive 模型上只看到 1 个或 2 个整数 N
→ 放大到某些 d 以后，可以“冒出” 9 个或 10 个整数 N
→ 但 rank 仍然还是 3 或 4
```

这正好解释了为什么本轮会出现：

- primitive `(70, 117)` 只有 `k = 1`
- 但它的若干放大版本已经变成 `K_9` 甚至 `K_10`
- 对应曲线的秩却一直停在 `3`

## 主结论 4：高 k 现在更像“整模型效应”，不太像“新的代数维度”

到 wl065 为止，最自然的读法已经变成：

```text
高 k = 同一条低秩椭圆曲线
       在不同整模型 / 不同放大倍数下
       出现了更多整数 N 表示
```

而不是：

```text
高 k = 椭圆曲线真的拥有更多独立基础解
```

这点对后面的反例搜索非常关键，因为它说明：

- 仅仅看到 `k` 很大，不代表代数自由度也很大
- 真正控制结构复杂度的，仍然更像是秩
- `closure` 为 0 的事实（wl063）与这里的“高 `k` 但低秩”是相容的

## 对 Harborth / Steiner-Beukers 方向的含义

把 wl063 和 wl065 合起来看：

```text
wl063: 338,225 个 G_M 顶点上 closure = 0
wl065: 最高阶 K_10 也只看到 rank = 3 或 4
```

于是当前 picture 更接近：

```text
反例之所以一直缺席，
不是因为高 k 顶点太少，
而是因为高 k 顶点大多仍被压缩在很低的秩里。
```

换句话说，`K_9 / K_10` 的出现并没有削弱“低秩障碍”这件事，反而强化了它。

## 没做的事

1. **把剩余 32 个 `K_9` 全跑完**
   - 目前只是 `10 / 42` 抽样。
   - 但从 10 个样本全部落在 `rank = 3 或 4` 看，继续全跑大概率只是把这个结论补齐。

2. **按 primitive 类去重后再做系统审计**
   - 现在已经看到“raw 顶点很多，但底型很少”。
   - 下一步更自然的是：
     - 先按 primitive `(A/g, B/g)` 归类
     - 再研究每个底型沿着不同放大倍数 `d` 的 `k(d)` 如何增长

3. **解释每个新增整数 N 的分母来源**
   - 现在只确认了“放大倍数会清分母，从而增加整数 `N`”。
   - 还没把这些新 `N` 在 primitive 曲线上对应到哪些有理平方点分母类型。

4. **`effort=2` 复核少数代表底型**
   - 目前 `effort=1` 都已 certified。
   - 若要进一步消除疑虑，可以只挑 4 个 primitive 底型各跑一次 `effort=2`。

## 元注

如果说 wl060 的结论是“`K_8` 也没有把秩顶上去”，那 wl065 则更进一步：

```text
连 K_10 都没有把秩顶上去，
而且很多 K_9 / K_10 其实来自同一批 primitive 低秩底型。
```

这意味着今后看 partner graph 里的高阶 hub，最好区分两层：

- **图论层**：`k`、degree、`K_n` 阶数
- **代数层**：去掉公共放大后的底型、对应曲线的秩

原始 `K_n` 很大，未必代表代数维度真的很大；很多时候，它只是在说：

> 同一条低秩曲线，被很多不同的整数模型重新实现了。
