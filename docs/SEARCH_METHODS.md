# 搜索方法速览

这份文档只回答一个问题：项目里几个搜索方法分别在干什么，应该怎么理解。

## 一、`parametric`

命令：

```bash
uv run python scripts/search.py parametric --scale 200
```

大白话解释：
- 先选一组本原勾股数
- 再选一个有理比例
- 生成候选点
- 检查这个点到四个顶点的距离里，有几个是有理数

优点：
- 最直观
- 三顶点结果很多
- 适合做基线验证

缺点：
- 搜索的是“参数范围”，不是“所有等价整数结构”
- 范围大了之后会慢

## 二、`ec`

命令：

```bash
uv run python scripts/search.py ec --max-m 30
```

它不是从零开始乱试，而是：
- 先从已知三顶点解里找 seed
- 再沿着椭圆曲线把 seed 扩展成更多有理解

可以把它理解成：
- `parametric` 更像“地毯式试”
- `ec` 更像“沿着一条有规律的轨道继续走”

优点：
- 能把一个 seed 扩展成很多相关解
- 更适合研究三顶点轨道结构

缺点：
- 需要 seed 才能启动
- 目前没有证据表明它能直接打通四顶点问题

## 三、`chain`

命令：

```bash
uv run python scripts/search.py chain --max-val 500
```

它不直接看坐标，而是改成找四个整数 `a,b,c,d`，要求：
- `a²+b²` 是平方
- `b²+c²` 是平方
- `c²+d²` 是平方
- `d²+a²` 是平方

这可以理解成：
- 四条边首尾连起来
- 相邻两条边都能拼成一个直角三角形

如果再加上 `a+c=b+d`，才会落回单位正方形问题。

所以：
- `chain` 默认是在找“长方形问题”
- 不是默认就在找 Harborth 主问题

## 四、`chain-fast`

命令：

```bash
uv run python scripts/search.py chain-fast --max-hyp 20000
```

这是当前最重要的主线。核心想法是：
- 不是枚举所有 `a,b,c,d`
- 而是只枚举两组本原勾股数
- 由这两组数直接推出候选结构
- 前两条条件自动满足，只检查后两条

这样复杂度被压成了更可控的 O(n²)。

特点：
- 直接针对正方形主问题
- 覆盖更完整
- 已支持 `numpy` 加速、near-miss、SQLite 持久化

## 五、`concordant`

命令：

```bash
uv run python scripts/search.py concordant --pair 264,420
```

这条线做的不是直接搜索点，而是分析：
- 对固定的 `(A,B)`
- 是否存在 `N`
- 使得 `N²+A²` 和 `N²+B²` 同时为平方

这和经典数论里的 concordant form / 椭圆曲线问题是同一个东西。

作用：
- 解释为什么有些“后两条成立”的半解会出现
- 帮助判断某个 `(A,B)` 对是否在数论上比较特殊

它不是当前主搜索器，更像分析工具。

## 六、怎么选方法

如果你的目标是：
- 看三顶点解长什么样：用 `parametric`
- 研究三顶点轨道结构：用 `ec`
- 看前四条勾股条件有哪些长方形解：用 `chain`
- 正面搜索 Harborth 主问题：用 `chain-fast`
- 分析某组 `(A,B)` 为什么会出现半解：用 `concordant`

## 七、当前最推荐的理解方式

- `parametric` 和 `ec`：偏“三顶点研究”
- `chain`：偏“长方形问题”
- `chain-fast`：偏“正方形主战场”
- `concordant`：偏“数论分析工具”

这样看项目会清楚很多，不容易把几条路线混在一起。
