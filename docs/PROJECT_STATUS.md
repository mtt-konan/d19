## 一、项目目标

项目研究的是 Harborth 猜想：

> 是否存在平面上一点，到单位正方形四个顶点的距离都为有理数？

项目里主要有两种等价视角：
- 坐标视角：直接找正方形里的有理点
- 整数视角：把问题改写成四个整数 `a,b,c,d` 的勾股条件

### 3. chain（长方形路线）

已经完成：
- 枚举 `a,b,c,d`，只要求四条边都能组成勾股对
- 默认不要求 `a+c=b+d`

这条线已经能稳定找到很多“长方形解”，也就是前四条成立，但不一定对应单位正方形。

已经完成：

- SQLite 持久化、near-miss、resume

这条线现在是项目里最重要的完整搜索器。

## 三、当前结论

到目前为止，最重要的结论是：
- 长方形解很多
- 四顶点正方形解仍然没找到
- “没找到”不像明显代码 bug，更像问题本身非常难

更具体一点：
- `chain` 可以很快找到很多前四条成立的解
- `chain-fast` 把正方形约束一起纳入后，目前在已搜索范围内仍是 0 解
- concordant form 分析说明：后两条单独成立并不罕见，但和前两条绑在一起就很硬

## 四、已经澄清的关键事实

### 1. 长方形问题和正方形问题不是一回事

只要求四条勾股关系成立时，能找到不少例子。正方形还多一个约束：

`a + c = b + d`

这个约束会把原本还有自由度的一条边直接定死，所以很多“半解”都会在这里断掉。

### 2. 交叉乘积族可以直接排除

如果 `(a,b,c,d) = (pm, qm, qn, pn)`，那么自动有 `ac = bd`，而且可证明永远不可能满足 `a+c=b+d`。这类结果对 Harborth 主问题没有帮助，已经在搜索里排除了。

### 3. 后两条成立，不代表前两条也会成立

像 `(264,315,420)`、`(264,352,420)` 这种例子说明：
- 可以找到一个 `d`，让 `c²+d²` 和 `d²+a²` 都是平方
- 但正方形约束会把 `b` 强行定成 `a+c-d`
- 这个被定死的 `b` 往往不能再让前两条同时成立

所以“半解”不少，但完整解极少甚至可能不存在。

## 五、当前主要瓶颈

数学瓶颈：
- 现在最缺的不是代码，而是更强的剪枝条件
- 已有过滤包括奇偶、mod 4、交叉乘积族排除，但还不够强

工程瓶颈：
- `chain-fast` 复杂度仍然是 O(n²)
- 范围扩大一倍，工作量大约变四倍
- `numpy` 能提速，但不能改变增长趋势
- `numpy` 快路径还受 `int64` 安全范围限制，`max_hyp` 超过大约 `36000` 后会回退到 Python 精确整数

## 六、下一阶段最值得做的事

优先级从高到低大致是：
- 找更强的数论剪枝，尽量少枚举无效三元组对
- 给 `chain-fast` 增加更强并行
- 用 concordant form / EC 工具分析哪些 `(A,B)` 对更值得重点看

简单说，项目已经从“先把程序写出来”进入“想办法把数学结构榨出来”的阶段。
